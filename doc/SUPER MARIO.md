## 1. 背景与原理科普

### 1.1 什么是 FC？

FC（Family Computer，俗称"红白机"，在欧美叫 NES）是任天堂 1983 年发售的家用游戏机。它没有硬盘和操作系统，游戏以**卡带（ROM）**形式插入。开机后，主机直接读取卡带里的程序并运行。

### 1.2 什么是"模拟器"？

模拟器（Emulator）就是用软件**逐个还原 FC 内部的硬件芯片的行为**，让原本只能在真实主机上跑的游戏程序，能在 PC 上运行。本质上是"用 Python 假装自己是一台红白机"。

游戏 ROM 文件（`.nes`）不认识它运行在真机还是模拟器上——只要模拟器对各芯片的模拟足够准确，游戏就能正常运行。

### 1.3 FC 的核心部件（也就是我们要模拟的对象）

| 部件 | 全称 | 作用 | 通俗类比 |
|------|------|------|----------|
| **CPU** | 中央处理器（6502 芯片） | 执行游戏逻辑：马里奥怎么跳、敌人怎么动、分数怎么算 | 大脑 |
| **PPU** | 图像处理器（Picture Processing Unit） | 把画面画到屏幕上：背景、马里奥、金币等 | 画师 |
| **APU** | 音频处理器（Audio Processing Unit） | 产生声音：背景音乐、跳跃音效、吃金币声 | 乐手 |
| **内存（RAM）** | — | 临时存放运行时数据 | 草稿纸 |
| **卡带 / Mapper** | 卡带 + 映射器 | 存放游戏程序和图像数据，Mapper 决定如何读取 | 书本 + 书签规则 |
| **手柄** | Controller | 接收玩家按键 | 遥控器 |

模拟器要做的，就是把上面这几块用代码实现出来，并让它们像真机一样协同工作。

### 1.4 这些部件如何协同（一帧画面是怎么来的）

FC 每秒刷新约 **60 帧**画面。每一帧里大致发生：

1. **CPU** 运行游戏逻辑，算出这一帧里所有角色的新位置、新状态，并把要显示的内容写进 PPU。
2. **PPU** 根据这些数据，把一整屏画面（256×240 像素）逐行画出来。
3. **APU** 同时产生这一帧对应的声音。
4. **手柄**的按键被 CPU 读取，影响下一帧的逻辑。

模拟器需要在 PC 上**循环重复**这个过程，并保证速度接近真机（约 60 次/秒），玩家才会觉得流畅、不卡顿、不变速。

### 1.5 为什么"只跑超级玛丽1代"能简化很多？

《超级马里奥兄弟》1代使用最简单的卡带类型 **NROM（Mapper 0）**——没有复杂的内存切换逻辑。这意味着我们可以**只实现 Mapper 0**，不必处理其他几十种映射器，大幅降低开发量。这也是本项目把目标聚焦在"1代"上的原因。

## 2. 项目目标

开发一个用 Python 编写、基于 Pygame 的 FC 模拟器，能够**正确、流畅地运行《超级马里奥兄弟》1代**，画面、操作、声音均接近原机体验，并附带基础调试工具用于学习与排错。

### 2.1 成功标准（验收标准）

- [ ] 能加载并运行《超级马里奥兄弟》1代 `.nes` ROM。
- [ ] 标题画面、关卡画面正确显示，无明显花屏。
- [ ] 马里奥可正常移动、跳跃、顶砖块、吃蘑菇、踩敌人、进水管、过关。
- [ ] 背景音乐与音效正常播放。
- [ ] 运行帧率接近 60 FPS，操作无明显延迟。
- [ ] 基础调试工具可用。

---

## 3. 功能需求

### 3.1 兼容范围

| 项 | 需求 |
|----|------|
| 目标游戏 | **仅需支持《超级马里奥兄弟》1代** |
| 卡带映射器 | 仅需实现 **NROM（Mapper 0）** |
| 其他游戏 | 不作要求（能跑则为额外收获，不作为验收项） |

> 说明：聚焦单一游戏与单一映射器，是为了把精力集中在把马里奥跑通跑好上，避免过度设计。

### 3.2 模拟核心部件

| 部件 | 需求 | 优先级 |
|------|------|--------|
| CPU（6502） | 完整实现马里奥所用到的全部指令与寻址方式 | 必须 |
| PPU（图像） | 背景、精灵（角色）、滚动、调色板，正确输出 256×240 画面 | 必须 |
| **APU（音频）** | **实现声音芯片，输出背景音乐与音效** | 必须 |
| 内存 / RAM | 实现 CPU 与 PPU 的内存映射 | 必须 |
| 卡带 / Mapper 0 | 解析 `.nes`（iNES 格式）文件，加载程序与图像数据 | 必须 |

> 关于音频（APU）：这是相对独立、有一定难度的模块。需求明确为**需要完整声音**（背景音乐 + 音效），因此 APU 列为必须项。

### 3.3 输入控制

| 项 | 需求 |
|----|------|
| 输入设备 | **仅键盘**（不要求支持游戏手柄） |
| 玩家人数 | **仅 1P（单人）**，不要求 2P |
| 按键映射 | 需将键盘按键映射到 FC 手柄的 8 个按钮：方向键 ↑↓←→、A、B、Start、Select |

**建议默认按键映射**（可在文档/代码中固定，后续可调整）：

| FC 手柄按钮 | 键盘按键（建议） |
|-------------|------------------|
| 方向 上/下/左/右 | ↑ / ↓ / ← / → |
| A（跳跃） | K |
| B（加速 / 发射火球） | J |
| Start（开始/暂停） | Enter |
| Select | 右 Shift |

> 注：以上为推荐方案，最终键位可在开发时确认。

### 3.4 画面显示

| 项 | 需求 |
|----|------|
| 原生分辨率 | FC 输出 256×240 像素 |
| 显示方式 | **固定倍数放大窗口**（如 2 倍 = 512×480，或 3 倍 = 768×720） |
| 全屏 | 不要求 |
| 图形库 | **Pygame** |

> FC 原生分辨率在现代屏幕上太小，因此按固定整数倍放大显示，保证像素清晰、不变形。

### 3.5 声音

| 项 | 需求 |
|----|------|
| 是否需要 | **需要完整声音** |
| 内容 | 背景音乐 + 各类音效（跳跃、吃金币、踩敌人、死亡等） |
| 实现 | 模拟 APU，通过 Pygame 音频输出 |

### 3.6 ROM 加载

| 项 | 需求 |
|----|------|
| 加载方式 | **命令行参数传入**，例如：`python main.py mario.nes` |
| ROM 来源 | **由用户自备**，模拟器**不内置**任何版权游戏 ROM |
| 文件格式 | iNES 格式（`.nes`） |

> 版权说明：本项目仅为模拟器程序本身，不分发、不内置任何任天堂游戏 ROM。用户需自行合法获取 ROM 文件。

### 3.7 存档

| 项 | 需求 |
|----|------|
| 即时存档（Save State） | **不需要**。每次运行从头开始即可。 |

### 3.8 调试工具（基础）

需要一套**基础调试工具**，用于学习 FC 原理与排查问题。建议至少包含：

| 调试功能 | 说明 | 优先级 |
|----------|------|--------|
| CPU 状态查看 | 显示寄存器（A、X、Y、PC、SP、状态标志）当前值 | 建议 |
| 内存查看 | 查看指定地址区间的内存内容（十六进制） | 建议 |
| 单步 / 帧步进 | 暂停后逐条指令或逐帧执行，观察变化 | 建议 |
| PPU 画面调试 | 查看图块（Pattern Table）、调色板等，便于排查花屏 | 可选 |
| 运行信息 | 显示当前 FPS、运行/暂停状态 | 建议 |

> 调试工具的具体形态（独立窗口 / 控制台输出 / 快捷键触发）可在开发阶段确定，不作硬性约束。

## 4. 非功能需求

### 4.1 性能

| 项 | 目标 |
|----|------|
| 帧率 | **接近原机 60 FPS** |
| 输入延迟 | 操作响应及时，无明显卡顿 |

> 说明：Python 是解释型语言，性能不如 C/C++。要达到 60 FPS 可能需要对热点代码（CPU、PPU 循环）做优化。马里奥1代逻辑相对简单，通常可以达成该目标；若个别环节吃力，可在保证正确性的前提下做针对性优化。

### 4.2 运行环境

| 项 | 要求 |
|----|------|
| 操作系统 | **以 Windows 为主**（当前为 Windows 11） |
| 跨平台 | 不强制要求（Pygame 本身跨平台，但不作为验收项） |
| Python 版本 | 建议 3.10 及以上 |
| 主要依赖 | Pygame（图形、输入、音频）；如需性能可选用 NumPy 等 |

### 4.3 代码与可维护性（建议）

- 各部件（CPU / PPU / APU / Mapper / 输入）模块清晰分离，便于理解与调试。
- 关键模块配注释，帮助学习 FC 原理。

## 5. 范围边界（明确不做的事）

为避免范围蔓延，以下内容**本期不做**：

- ❌ 其他映射器（MMC1、MMC3 等）与其他游戏的兼容。
- ❌ 游戏手柄（仅键盘）。
- ❌ 2P 双人。
- ❌ 即时存档 / 读档。
- ❌ 全屏模式、可变缩放、滤镜（仅固定倍数窗口）。
- ❌ 图形界面菜单 / ROM 选择器（仅命令行传入 ROM）。
- ❌ 跨平台打包发布。

> 这些项目均为后续可扩展方向，不在本期目标内。
